# iPhone 捷徑設定指南 — KidsChatApp

**目標**：對著 iPhone 或 HomePod 說「嘿 Siri，小老師」，
觸發語音對話，並將 AI 的回覆用 Siri 語音唸出來。

---

## 前置條件

- Mac 或 Pi5 上的 KidsChatApp server 已啟動
- iPhone 與 server **在同一個 Wi-Fi**
- 已取得 server 的區網 IP（例：`192.168.1.105`）

取得 IP 指令（在 Mac 上）：

```bash
ipconfig getifaddr en0
```

---

## 捷徑步驟（逐一操作）

打開 iPhone「捷徑」App → 右上角「＋」新增捷徑

---

### 步驟 1 — 接收語音輸入

| 動作 | 設定值 |
|------|--------|
| 搜尋「要求輸入」 | 類型：**文字** |
| 提示語 | `說話給小老師聽` |
| 將結果儲存為 | `使用者說的話` |

---

### 步驟 2 — 呼叫 API

| 動作 | 設定值 |
|------|--------|
| 搜尋「取得 URL 的內容」 | |
| URL | `http://192.168.1.105:8000/chat` ← **換成你的 IP** |
| 方法 | **POST** |
| 標頭 | `Content-Type` = `application/json` |
| 要求內文 | **JSON** |

JSON 內文結構（點「＋」逐行新增）：

```json
{
  "user_id": "kid_01",
  "message": [使用者說的話 變數],
  "mode": "chat"
}
```

> **重要**：`message` 的值要從「變數」選「使用者說的話」，
> 不是手動輸入文字。

---

### 步驟 3 — 解析回應

| 動作 | 設定值 |
|------|--------|
| 搜尋「從字典取得值」 | |
| 取得 | `reply` |
| 輸入 | `URL 的內容`（上一步的結果） |
| 儲存為 | `AI回覆` |

---

### 步驟 4 — Siri 語音播報

| 動作 | 設定值 |
|------|--------|
| 搜尋「朗讀文字」| |
| 文字 | 選變數 `AI回覆` |

---

### 步驟 5 — 儲存並設定 Siri

1. 捷徑右上角 `⋯` → 「加入 Siri」
2. 錄製短語：「**小老師**」
3. 儲存

---

## 測試流程

```
對 iPhone/HomePod 說：「嘿 Siri，小老師」
         ↓
Siri 詢問：「說話給小老師聽」
         ↓
你說：「我是小公主」
         ↓
iPhone 呼叫 http://192.168.1.105:8000/chat
         ↓
Siri 朗讀：「哇！原來是漂亮的小公主～要一起唱歌嗎？」
```

---

## 進階：自動連續對話版

若要讓對話持續（不需每次喚醒 Siri），在步驟 4 之後加入：

| 動作 | 設定值 |
|------|--------|
| 「重複」 | 直到「取得 URL 的內容」失敗，或加入「停止」按鈕 |

> **提示**：可在捷徑開頭加「如果 Wi-Fi 連線 不等於 你家 SSID，停止執行」，
> 防止在外面意外觸發。

---

## 模式切換（進階用法）

在 JSON 內文的 `mode` 欄位換值，可切換互動模式：

| mode 值 | 說明 |
|---------|------|
| `chat` | 一般對話（預設） |
| `sing` | 唱兒歌模式 |
| `story` | 講故事模式 |
| `roleplay` | 角色扮演模式 |

---

## 重置對話（清除歷史）

對 Siri 說出捷徑，或用 curl：

```bash
curl -X DELETE http://192.168.1.105:8000/session/kid_01
```

---

## 常見問題

**Q: 捷徑顯示「無法連線」**
A: 確認 server 正在執行、IP 正確、手機在同一 Wi-Fi。
   測試：`curl http://192.168.1.105:8000/health`

**Q: Siri 唸出 JSON 原始文字**
A: 「從字典取得值」的步驟設定有誤，確認 key 是 `reply`。

**Q: HomePod 無法觸發**
A: HomePod 需設定為「家庭中樞」，且捷徑必須已設好 Siri 短語，
   並在「家庭」App 中授權。
